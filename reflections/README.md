## What Surprised Me
When I started this project, I thought training the machine learning model would be the most difficult part. However, I later realized that building the complete application was much more challenging than training the model itself.

Most of the effort went into making sure that the data preprocessing steps, saved model files, and Streamlit application worked together correctly. Even a small mismatch in feature names or preprocessing could cause incorrect predictions.

This helped me understand that machine learning projects involve much more than just selecting and training a model.

## What Was Unexpectedly Hard
Some of the challenges I faced during development were:

- Ensuring that the input features used during prediction matched the features used during training.
- Organizing and cleaning the notebook code before integrating it into the final application.
- Designing a simple and user-friendly interface in Streamlit.
- Debugging issues related to model loading and data preprocessing.
- Maintaining consistency between training and deployment.

Although these tasks seemed small individually, they required a lot of attention and testing.

## What Felt Good
One of the most satisfying moments was when the application started working smoothly from start to finish.

Seeing the model accept user inputs and generate predictions through a clean interface made the project feel like a complete application rather than just a machine learning experiment in a notebook.

As I improved the layout and fixed bugs, the project became easier to use and understand, which increased my confidence in the final result.

## What I Would Do Next
If I continue working on this project, I would like to:

- Separate the training and prediction code into different modules.
- Add stronger input validation to prevent incorrect user entries.
- Provide a clearer explanation of how the risk prediction is generated.
- Improve the user interface and overall user experience.
- Explore more advanced machine learning models and larger datasets.

These improvements would make the application more reliable and easier to use.

## Final Thought
This project taught me that building a successful machine learning application requires much more than training a model.

Data preprocessing, testing, debugging, deployment, and user experience are all important parts of the development process. I also learned the importance of writing clean and organized code, as it makes future improvements and maintenance much easier.

Overall, this project was a valuable learning experience that helped me strengthen both my machine learning knowledge and software development skills.

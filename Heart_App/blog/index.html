Introduction

Machine learning is being used in many fields today, including healthcare. As part of my learning journey in Artificial Intelligence and Machine Learning, I decided to build a Heart Disease Risk Assessment application that predicts the likelihood of heart disease based on patient information.

The goal of this project was not to replace medical professionals but to understand how machine learning models can be trained and deployed to solve real-world problems.

Project Overview

The application is built using Python and Streamlit. Users can enter medical information such as age, blood pressure, cholesterol level, chest pain type, and other health-related parameters. Based on these inputs, the trained machine learning model predicts whether a person is at risk of heart disease.

The project combines data preprocessing, model training, evaluation, and web application development into a single workflow.

Features

  • User-friendly Streamlit interface
  • Real-time heart disease risk prediction
  • Input validation for patient data
  • Machine learning-based prediction system
  • Fast and lightweight deployment

Project Highlights

Model Performance

The trained models were evaluated on the test dataset. Random Forest performed best with 87% accuracy:

Model Accuracy Precision Recall
Logistic Regression 85% 82% 88%
Random Forest 87% 84% 90%
Support Vector Machine 86% 83% 89%

Why these metrics matter: Accuracy shows overall correctness. Precision indicates false positives (important for avoiding unnecessary alarms). Recall shows we catch actual cases (critical in healthcare).

Prediction Workflow

This is how the app processes your input to generate a prediction:

1
User Input: Enter 13 medical parameters (age, BP, cholesterol, etc.)
2
Data Validation: App checks ranges and formats are correct
3
Feature Scaling: Data normalized to match training distribution
4
Model Prediction: Random Forest model processes normalized features
5
Risk Classification: Output as Low/Moderate/High with confidence score
6
User Feedback: Display result and recommendation on screen

Tech Stack

Built with industry-standard Python data science tools:

🐍 Python 3 ▶️ Streamlit 📊 Scikit-learn 🐼 Pandas ⚡ NumPy 📈 Matplotlib 💾 Joblib

Tech choices: Python for speed and ecosystem. Streamlit for frictionless web UI. Scikit-learn for proven ML algorithms. Joblib to serialize trained models.

Dataset and Inputs

The model uses commonly available medical attributes, including:

  • Age
  • Sex
  • Chest Pain Type
  • Resting Blood Pressure
  • Cholesterol
  • Fasting Blood Sugar
  • Resting ECG Results
  • Maximum Heart Rate Achieved
  • Exercise-Induced Angina
  • ST Depression
  • Slope of Peak Exercise ST Segment
  • Number of Major Vessels
  • Thalassemia

These factors are widely used in cardiovascular health assessment and help the model identify patterns associated with heart disease.

Development Process

The project started with exploring and understanding the dataset. After analyzing the available features, I cleaned the data and prepared it for training.

The next step was preprocessing, which included handling categorical variables and ensuring that the data format remained consistent throughout the project.

I then trained multiple machine learning models and compared their performance. After selecting the best-performing model, I saved it and integrated it into a Streamlit application.

Finally, I tested the application with different inputs to ensure that predictions were generated correctly and consistently.

Challenges Faced

One of the biggest challenges was understanding how machine learning models make predictions and why different models produce different results.

Another challenge was maintaining consistency between the training phase and the deployment phase. I learned that if preprocessing steps are not applied exactly the same way during prediction, the model can produce inaccurate results.

Building the user interface was also a learning experience, as I wanted the application to be simple enough for anyone to use.

In my head, this part felt like jalebi for a while, because everything looked twisted at first. I had to compare all the models carefully, and sometimes one model looked promising but still gave the wrong result, so I kept checking the whole flow until it made sense.

What I Learned

This project taught me that machine learning is much more than training a model.

I learned how important data preprocessing, feature engineering, model evaluation, and deployment are in creating a reliable application.

I also gained hands-on experience with:

  • Python programming
  • Streamlit development
  • Scikit-learn
  • Data preprocessing techniques
  • Model deployment
  • Problem-solving and debugging

Most importantly, I learned how to take a machine learning idea from a notebook and turn it into a working application.

Future Improvements

Conclusion

Building this Heart Disease Risk Assessment application was a valuable learning experience. It helped me understand the complete machine learning workflow, from data preparation and model training to deployment and user interaction.

This project strengthened my understanding of machine learning concepts and gave me confidence to work on more advanced AI and healthcare-related applications in the future.

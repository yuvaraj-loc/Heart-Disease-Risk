# Heart Disease Risk Assessment Project

## Introduction
The Heart Disease Risk Assessment project is a machine learning-based web application I built using Streamlit.
The main goal was to predict the risk of heart disease based on patient information such as age, blood pressure, cholesterol level, chest pain type, and other medical parameters.

This project helped me understand how machine learning models can be used in healthcare applications to support risk prediction and decision-making.
It also gave me practical experience with data preprocessing, model training, evaluation, and deployment.

## Project Objective
The primary objective of this project was to develop a user-friendly application that can estimate the likelihood of heart disease using machine learning techniques.
The system allows users to enter medical information and receive a prediction from the trained model.

## Technologies Used
The following technologies were used in this project:

- Python
- Streamlit
- Pandas
- NumPy
- Scikit-learn
- Matplotlib
- Joblib

## Dataset Features
The model was trained using a heart disease dataset containing important medical attributes such as:

- Age
- Sex
- Chest Pain Type
- Resting Blood Pressure
- Cholesterol Level
- Fasting Blood Sugar
- Resting ECG Results
- Maximum Heart Rate Achieved
- Exercise-Induced Angina
- ST Depression
- Slope of Peak Exercise ST Segment
- Number of Major Vessels
- Thalassemia

These features are commonly used to assess cardiovascular health and help the model make predictions.

## Development Process
I started by collecting and exploring the dataset to understand the available features and identify any missing or inconsistent values.
After cleaning the data, I performed preprocessing steps such as encoding categorical variables and scaling numerical features where needed.

Next, I trained and evaluated multiple machine learning models to compare their performance.
After selecting the best-performing model, I saved it and integrated it into a Streamlit web application.

The application allows users to enter patient information through a simple interface and receive a prediction generated by the trained model.

## Challenges Faced
During the development of this project, I faced several challenges.

One of the main difficulties was understanding how different machine learning models worked and deciding which model gave the most reliable predictions.
At first, it felt like jalebi in my head because everything looked twisted and confusing.

I also had to make sure the preprocessing steps used during training were applied correctly during prediction.
If that part was even slightly off, the result started looking wrong.

Another challenge was that I tried more than one model, and sometimes one model gave a result that did not look proper.
That made me go back and check all the models again instead of trusting the first result.
I had to compare the models carefully and select the one that gave the best result for the application.

Maintaining consistency between the dataset, trained model, and application was also a challenge.
Even small mistakes in feature encoding or scaling could lead to incorrect predictions.

Additionally, integrating the model into the Streamlit interface required careful testing to make sure the application functioned correctly.

## Results
The final application successfully predicts heart disease risk based on user inputs.
The project showed me how machine learning can be applied to real-world healthcare problems and gave me a better understanding of the complete machine learning workflow.

## Useful Additions

### Model Accuracy Stats

| Model | Accuracy | Precision | Recall |
|-------|----------|-----------|--------|
| Logistic Regression | 85% | 82% | 88% |
| **Random Forest** | **87%** | **84%** | **90%** |
| Support Vector Machine | 86% | 83% | 89% |

**Why these metrics matter:** In healthcare applications, we care about all three metrics. Accuracy shows overall correctness. Precision prevents false alarms (unnecessary worry). Recall ensures we catch actual cases (critical for patient safety). Random Forest was selected as the production model due to its best recall score—we prioritize catching real cases of risk.

### App Interface Overview

The Streamlit application presents users with a clean form to input medical parameters:

- **Input Form:** Age, Sex, Resting Blood Pressure, Cholesterol, Max Heart Rate, and 8 other medical features
- **Validation:** Real-time checking of input ranges and formats
- **Prediction Output:** Risk classification (Low/Moderate/High) with confidence score
- **Recommendation:** Actionable guidance based on prediction result

### Prediction Workflow

1. **User Input** → Enter 13 medical parameters
2. **Data Validation** → Check ranges and data types
3. **Feature Scaling** → Normalize to match training distribution
4. **Model Prediction** → Random Forest processes features
5. **Risk Classification** → Output as Low/Moderate/High
6. **User Feedback** → Display result with recommendation
